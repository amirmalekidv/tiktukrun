🐛 جدول کامل باگ‌ها (۲۹ باگ)

# ماژول شرح باگ شدت وضعیت راه‌حل
۱Build / SWC ۷۵۸ خطای TypeScript در زمان compile با tsc — جلوگیری از build کامل بحرانی✅ رفع‌شد گذار از tsc به swc: pnpm nest build --builder swc
۲Database Schema ۱۰ جدول گمشده در دیتابیس (chat_reports، user_segments، user_accessories، campaign_recipients، crm_notes، free_tickets، xp_history، customer_segments، roles، invite_codes) بحرانی✅ رفع‌شد pnpm prisma db push --accept-data-loss — ۵۳ جدول هم‌گام شد
۳Auth / Admin ورود ادمین با phoneNumber کار نمی‌کرد — Passport LocalStrategy فیلد mobile را تنها‌پذیر کرده بود بحرانی✅ رفع‌شد تأیید usernameField='mobile' در local.strategy.ts
۴Auth / Admin کاربر ادمین در دیتابیس passwordHash نداشت → 401 در ورود بحرانی✅ رفع‌شد اضافه کردن ستون و bcrypt-hash کردن رمز ادمین Admin@123456
۵Analytics Service استفاده از booking.totalPrice در ۸ کوئری aggregate در حالی که فیلد اسم totalAmount دارد → 500 بحرانی✅ رفع‌شد بازنویسی کامل analytics.service.ts + bump cache key به :v2
۶Analytics Service کوئری‌های $queryRaw با نام جدول PascalCase (User، Booking، Game، Review، Category) به جای snake_case بحرانی✅ رفع‌شد تبدیل به نام‌های واقعی PostgreSQL + پوشش با .catch(() => [])
۷Customers Service User.isVip / totalSpent / lastActiveAt روی مدل User استفاده شده در حالی که این فیلدها روی UserProfile هستند → 500 بحرانی✅ رفع‌شد بازنویسی customers.service.ts با orderBy.profile.{totalSpent,levelId}
۸Customers Service ساخت مشتری از {name, phone} استفاده می‌کرد در حالی که schema {fullName, mobile, inviteCode} دارد بحرانی✅ رفع‌شد اصلاح فیلدها در createCustomer
۹CRM Notes استفاده از CrmNote.text و createdById در حالی که schema body و authorId دارد متوسط✅ رفع‌شد اصلاح نام فیلدها
۱۰Campaigns Service Segment.memberCount در دیتابیس وجود ندارد — اسم واقعی cachedCount بحرانی✅ رفع‌شد تغییر در campaigns + segments service
۱۱Campaigns Service Campaign.clickCount در schema نیست — اسم درست clickedCount متوسط✅ رفع‌شد اصلاح در aggregate و orderBy
۱۲Users Admin route /admin/users/stats قبل از /admin/users/:id قرار نگرفته بود → 404 / تطبیق با :id="stats" → 500 بحرانی✅ رفع‌شد جابه‌جایی ترتیب @Get('stats') قبل از @Get(':id')
۱۳Users Admin متد adminGetStats() پیاده‌سازی نشده بود متوسط✅ رفع‌شد افزودن متد با ۷ aggregate count (total / active / banned / muted / new7 / new30 / withProfile)
۱۴Chat Service کوئری raw از جدول "ChatMessage" در حالی که PostgreSQL جدول را chat_messages دارد → 500 در peakHour متوسط✅ رفع‌شد اصلاح نام جدول + try/catch محافظ
۱۵Tickets Service جدول "Ticket"/"TicketMessage" به جای tickets/ticket_messages + ستون isStaff به جای isStaffReply متوسط✅ رفع‌شد اصلاح کامل raw SQL
۱۶Tickets Service استفاده از enum مقدار WAITING_STAFF که در schema نیست — مقادیر واقعی IN_PROGRESS / WAITING_USER متوسط✅ رفع‌شد اصلاح آرایه status
۱۷Wheel Admin orderBy.createdAt روی WheelSpin که فیلد awardedAt دارد + _sum.paidAmount به جای costPaid متوسط✅ رفع‌شد اصلاح نام فیلدها + Number(userId) برای فیلتر
۱۸Avatar Service {sortOrder: 'asc'} در حالی که schema requiredLevel دارد + isEquipped به جای isActive متوسط✅ رفع‌شد sed اصلاح نام‌های فیلد در سراسر فایل
۱۹Cities CRUD سرویس id: string به findUnique({where:{id}}) پاس می‌داد در حالی که schema Int است → "Expected Int, provided String" بحرانی✅ رفع‌شد افزودن helper toIntId() + تبدیل در همه CRUD methods
۲۰Categories CRUD فیلد color در DTO اختیاری بود ولی در schema الزامی → Prisma "Argument color is missing" بحرانی✅ رفع‌شد افزودن مقدار پیش‌فرض #8B0000 + اصلاح id type
۲۱Branches CRUD DTO cityId: string ولی schema Int → خطای validation بحرانی✅ رفع‌شد تبدیل cityId و managerId به Int در create/update
۲۲Discount Codes CRUD DiscountCode.value در schema Int است اما کد BigInt(...) پاس می‌داد + فیلد gameIds یک relation است نه scalar بحرانی✅ رفع‌شد بازنویسی discounts.service.ts با Int + حذف gameIds از data
۲۳Discount Usages فیلد رابطه‌ای discountCodeId در حالی که schema آن را codeId نام‌گذاری کرده متوسط✅ رفع‌شد اصلاح به codeId
۲۴Segments CRUD فیلدهای color و icon در schema الزامی + memberCount به جای cachedCount بحرانی✅ رفع‌شد افزودن defaults + اصلاح تمام نام‌های فیلد + Int id
۲۵Segments Evaluator user.level include — level روی UserProfile است نه User متوسط✅ رفع‌شد تغییر include به profile.level + segmentId به Int
۲۶Games / Sections section featured در نقشه نبود → 400 + orderBy: {totalBookings:'desc'} در حالی که این فیلد روی Game وجود ندارد → 500 بحرانی✅ رفع‌شد افزودن featured/stories/leaderboard/paintball + تغییر orderBy به siteRank
۲۷Games / Availability gameId: string به فیلتر game.id (Int) پاس می‌شد → 500 متوسط✅ رفع‌شد تبدیل Number(gameId)
۲۸Web Pages (Next.js 14) صفحات /games/[slug], /invite/[code], /section/[slug], /games/[slug]/booking از use(params) hook استفاده می‌کردند (الگوی Next 15) که در Next 14.2.13 پشتیبانی نمی‌شود → 500 بحرانی✅ رفع‌شد حذف use(params) و destructure مستقیم params: {slug: string}
۲۹Admin Page / Calendar صفحه /bookings/calendar از currentMonth.jDaysInMonth() استفاده می‌کرد — این static method در moment-jalaali است نه instance method → 500 متوسط✅ رفع‌شد محاسبه از طریق endOf('jMonth').format('jD')
