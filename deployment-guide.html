۲. کلون و نصب پکیج‌ها

سورس پروژه را در مسیر مناسب کلون کن. ما /opt/tiktakrun را پیشنهاد می‌دهیم:

# ساخت دایرکتوری و کپی سورس
sudo mkdir -p /opt/tiktakrun
sudo chown -R $USER:$USER /opt/tiktakrun
cd /opt/tiktakrun

# اگر zip داری، استخراج کن
unzip tiktakrun-FINAL-v1.0.0.zip -d .

# نصب همه پکیج‌های monorepo
pnpm install

# تولید Prisma Client
cd apps/api
pnpm prisma generate
cd ../..
💡 نکته: این پروژه یک monorepo با ساختار pnpm workspaces + Turborepo است. دایرکتوری‌های اصلی: apps/api (NestJS) ، apps/web (Next.js) ، apps/admin (Next.js)

⚠️ حیاتی: JWT_SECRET و JWT_REFRESH_SECRET را با مقادیر تصادفی طولانی (حداقل ۶۴ کاراکتر) جایگزین کن. می‌توانی با این فرمان تولیدشان کنی:
openssl rand -base64 64
📌 یادآوری: در apps/api/main.ts، API پورت را از متغیر PORT می‌خواند نه API_PORT. پس حتماً هر دو را با مقدار یکسان تنظیم کن.

۵. مهاجرت دیتابیس و سید کردن

بعد از بالا آمدن PostgreSQL، باید جداول را بسازی و داده‌های اولیه را وارد کنی:

cd /opt/tiktakrun/apps/api

# روش ۱ (پیشنهادی): با migration files

# روش ۲ (اگر migration نداری): مستقیم schema را push کن
pnpm prisma db push

# سید کردن داده‌های اولیه (شهرها، دسته‌ها، بازی‌ها، یوزر admin، ...)
pnpm prisma db seed
✅ بعد از سید: یک یوزر admin با مشخصات زیر ساخته می‌شود:
Mobile: 09120000000   Password: Admin@1234
⚠️ امنیت: در محیط محصول، بلافاصله بعد از اولین ورود، رمز admin را عوض کن:
۱. وارد /admin شو
۲. از منوی پروفایل، تغییر رمز عبور را انتخاب کن
۳. یا با psql دستی hash جدید بساز: 
node -e "console.log(require('bcrypt').hashSync('NewStrongPass!', 10))"
# سپس:
UPDATE users SET "passwordHash" = '$2b$10$...' WHERE mobile = '09120000000';

۹. تنظیم Nginx به‌عنوان Reverse Proxy

فرض می‌کنیم سه ساب‌دامین داری: tiktakrun.ir (وب)، admin.tiktakrun.ir (پنل)، و api.tiktakrun.ir (API).

فایل /etc/nginx/sites-available/tiktakrun

# ─────────── WEB (Frontend اصلی) ───────────
server {
    listen 80;
    server_name tiktakrun.ir www.tiktakrun.ir;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

# ─────────── ADMIN ───────────
server {
    listen 80;
    server_name admin.tiktakrun.ir;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# ─────────── API ───────────
server {
    listen 80;
    server_name api.tiktakrun.ir;
    client_max_body_size 20M;

    location / {
        proxy_pass http://localhost:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Socket.io
    location /socket.io/ {
        proxy_pass http://localhost:4000/socket.io/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}

فعال‌سازی

sudo ln -s /etc/nginx/sites-available/tiktakrun /etc/nginx/sites-enabled/
sudo nginx -t            # تست syntax
sudo systemctl reload nginx

۱۰. SSL با Certbot و Let's Encrypt

پیش‌نیاز: DNS رکوردهای A تمام ساب‌دامین‌ها باید به IP سرور اشاره کنند و در دسترس باشند. 
# دریافت گواهی برای همه ساب‌دامین‌ها
sudo certbot --nginx \
  -d tiktakrun.ir \
  -d www.tiktakrun.ir \
  -d admin.tiktakrun.ir \
  -d api.tiktakrun.ir

# Certbot به‌طور خودکار:
#  1. گواهی می‌گیرد
#  2. فایل Nginx را آپدیت می‌کند (HTTPS اضافه می‌کند)
#  3. redirect از HTTP به HTTPS را اعمال می‌کند

# تست تمدید خودکار
sudo certbot renew --dry-run

# Cron از قبل ساخته شده ولی برای اطمینان:
sudo systemctl status certbot.timer
✅ تمدید خودکار: Certbot هر روز چک می‌کند و در صورت لزوم تمدید می‌کند. گواهی‌های Let's Encrypt هر ۹۰ روز منقضی می‌شوند ولی ۳۰ روز قبل تمدید می‌شوند.

۱۱. پشتیبان‌گیری خودکار با Cron

روش ۱: استفاده از endpoint داخلی Admin

# این endpoint از داخل admin panel هم در دسترس است:
# POST /api/v1/admin/backup/create

# اسکریپت bash برای فراخوانی خودکار:
sudo nano /opt/tiktakrun/scripts/backup.sh

محتوای /opt/tiktakrun/scripts/backup.sh:

#!/bin/bash
set -e
DATE=$(date +%Y-%m-%d_%H-%M-%S)
BACKUP_DIR=/opt/tiktakrun/backups
mkdir -p $BACKUP_DIR

# Backup PostgreSQL
docker compose -f /opt/tiktakrun/docker-compose.yml exec -T postgres \
  pg_dump -U tiktakrun_user -d tiktakrun_db \
  | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Backup storage (uploaded files)
tar -czf $BACKUP_DIR/storage_$DATE.tar.gz -C /opt/tiktakrun storage/

# حذف بکاپ‌های قدیمی‌تر از ۳۰ روز
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete

echo "[$(date)] Backup completed: $DATE" >> /var/log/tiktakrun/backup.log
sudo chmod +x /opt/tiktakrun/scripts/backup.sh

# تست دستی
/opt/tiktakrun/scripts/backup.sh

# اضافه به crontab (هر روز ساعت ۳ بامداد)
crontab -e
# خط زیر را اضافه کن:
0 3 * * * /opt/tiktakrun/scripts/backup.sh

روش ۲: فراخوانی API endpoint

#!/bin/bash
TOKEN=$(curl -s -X POST https://api.tiktakrun.ir/api/v1/auth/admin/login \
  -H "Content-Type: application/json" \
  -d '{"mobile":"09120000000","password":"YourAdminPass"}' \
  | jq -r '.access_token')

curl -X POST https://api.tiktakrun.ir/api/v1/admin/backup/create \
  -H "Authorization: Bearer $TOKEN"

بازیابی از بکاپ

cd /opt/tiktakrun/backups
gunzip -c db_2026-05-26_03-00-00.sql.gz | \
  docker compose -f ../docker-compose.yml exec -T postgres \
  psql -U tiktakrun_user -d tiktakrun_db

۱۲. عیب‌یابی و رفع مشکلات رایج

🚨 مشکل ۱: API استارت نمی‌شود

چک‌لیست:

  • آیا PostgreSQL اجرا است؟ docker compose ps
  • آیا DATABASE_URL در .env صحیح است؟
  • آیا PORT و API_PORT هر دو ست شده‌اند؟
  • لاگ‌ها را ببین: pm2 logs tiktakrun-api --nostream
🚨 مشکل ۲: خطای Prisma client
cd /opt/tiktakrun/apps/api
pnpm prisma generate
pm2 restart tiktakrun-api
🚨 مشکل ۳: Web/Admin به API وصل نمی‌شود

بررسی NEXT_PUBLIC_API_URL در .env.local:

  • باید با /api/v1 پایان یابد (نه فقط /api)
  • بعد از تغییر، باید مجدد pnpm build اجرا کنی (Next.js در زمان بیلد متغیرها را inline می‌کند)
🚨 مشکل ۴: OTP پیامک نمی‌رسد

در محیط dev، کد OTP در لاگ‌های API چاپ می‌شود:

tail -f /var/log/tiktakrun/api-out.log | grep "SMS-MOCK OTP"

در محیط prod، باید SMS.ir credentials واقعی در .env ست شوند.

🚨 مشکل ۵: خطای bcrypt

اگر هنگام لاگین خطای bcrypt دیدی:

cd /opt/tiktakrun/apps/api
pnpm rebuild bcrypt
pm2 restart tiktakrun-api
🚨 مشکل ۶: WebSocket کار نمی‌کند

چک کن که Nginx بلاک location /socket.io/ دارد و headerهای Upgrade و Connection ست هستند.

🚨 مشکل ۷: Memory leak یا کندی
pm2 monit                          # مانیتورینگ زنده
pm2 restart all                    # ری‌استارت همه
docker compose restart postgres    # ری‌استارت دیتابیس
🚨 مشکل ۸: تداخل پورت با سرویس‌های دیگر

اگر پورت‌های ۳۰۰۰/۴۰۰۰/۵۴۳۲/۶۳۷۹ روی سرور قبلاً استفاده می‌شوند، در .env و docker-compose.yml پورت‌ها را تغییر بده. مثال (همان پورت‌های QA که در این session استفاده شد):

  • API: 4010
  • Web: 3010
  • Admin: 3011
  • PostgreSQL: 5433
  • Redis: 6380
