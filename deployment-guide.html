۲. کلون و نصب پکیج‌ها

سورس پروژه را در مسیر مناسب مستقر کن. ما /opt/tiktakrun را پیشنهاد می‌دهیم:

# ساخت دایرکتوری و انتقال مالکیت
sudo mkdir -p /opt/tiktakrun
sudo chown -R $USER:$USER /opt/tiktakrun
cd /opt/tiktakrun

# استخراج فایل zip تحویلی
unzip tiktakrun-FINAL-v1.0.0.zip -d .

# نصب همه پکیج‌های monorepo (pnpm workspaces)
pnpm install

# تولید Prisma Client برای MongoDB
cd apps/api
pnpm prisma generate
cd ../..
توجه: بعد از هر بار pnpm install یا تغییر در schema.prisma، حتماً prisma generate را دوباره اجرا کن، وگرنه TypeScript خطا می‌دهد.

۳. تنظیمات محیط (.env)

فایل .env را از روی .env.example بساز و مقادیر را پر کن:

cp .env.example .env

📄 محتوای کلیدی .env

# ---------- MongoDB ----------
MONGO_USER=tiktakrun
MONGO_PASSWORD=یک-رمز-بسیار-قوی-اینجا
MONGO_DB=tiktakrun_db

# رشته اتصال — توجه به replicaSet=rs0 و directConnection=true
DATABASE_URL="mongodb://tiktakrun:یک-رمز-بسیار-قوی-اینجا@mongo:27017/tiktakrun_db?authSource=admin&replicaSet=rs0&directConnection=true"

# ---------- Redis ----------

REDIS_PORT=6379

# ---------- API ----------
PORT=4000
NODE_ENV=production
JWT_ACCESS_SECRET=یک-سکرت-تصادفی-طولانی
JWT_REFRESH_SECRET=یک-سکرت-تصادفی-دیگر
JWT_ACCESS_EXPIRES=15m

# ---------- Frontend public URLs ----------

هشدار امنیتی: فایل .env هرگز نباید در گیت commit شود. مطمئن شو در .gitignore قرار دارد. برای تولید سکرت تصادفی از openssl rand -hex 32 استفاده کن.
چرا replicaSet=rs0؟ Prisma برای انجام تراکنش‌ها (Transactions) روی MongoDB به Replica Set نیاز دارد — حتی اگر تک‌نودی باشد. به همین خاطر در رشته اتصال replicaSet=rs0 اجباری است.

۴. راه‌اندازی MongoDB (Replica Set) و Redis

پلتفرم از MongoDB به‌صورت Replica Set تک‌نودی (rs0) استفاده می‌کند چون Prisma برای تراکنش‌ها به آن نیاز دارد. تمام تنظیمات در docker-compose.yml آماده است.

گام ۱ — ساخت Keyfile احراز هویت داخلی

اعضای Replica Set برای احراز هویت داخلی به یک keyfile مشترک نیاز دارند:

# تولید keyfile (اگر از قبل در سورس نیست)
openssl rand -base64 756 > infra/mongo/mongo-keyfile
chmod 400 infra/mongo/mongo-keyfile

# مالکیت برای کاربر mongodb داخل کانتینر (uid 999)
sudo chown 999:999 infra/mongo/mongo-keyfile
گام ۲ — بالا آوردن سرویس‌ها
# فقط mongo، mongo-init و redis را اجرا کن
docker compose up -d mongo redis

# سرویس mongo-init به‌صورت خودکار rs.initiate() را اجرا می‌کند
docker compose up mongo-init

# بررسی وضعیت Replica Set
docker compose exec mongo mongosh --eval "rs.status().myState"
# خروجی 1 یعنی PRIMARY و سالم است
راه‌اندازی خودکار: سرویس mongo-init اسکریپت infra/mongo/init-replica.sh را اجرا می‌کند که منتظر آماده شدن مونگو می‌ماند، سپس rs.initiate({_id:'rs0', members:[{_id:0, host:'mongo:27017'}]}) را صدا می‌زند و تا رسیدن به وضعیت PRIMARY صبر می‌کند.

راه‌اندازی دستی Replica Set (در صورت نیاز)

docker compose exec mongo mongosh -u tiktakrun -p --authenticationDatabase admin

# داخل shell مونگو:
rs.initiate({ _id: "rs0", members: [{ _id: 0, host: "mongo:27017" }] })
rs.status()

۵. اعمال Schema و سید کردن دیتابیس

مهم: MongoDB از prisma migrate پشتیبانی نمی‌کند. به‌جای آن از prisma db push استفاده کن که Schema را مستقیماً روی دیتابیس اعمال می‌کند.
cd apps/api

# اعمال schema روی MongoDB
pnpm prisma db push

# سید کردن داده‌های اولیه (دسته‌بندی‌ها، نقش‌ها، ادمین پیش‌فرض)
pnpm prisma db seed
# یا مستقیم:
npx tsx prisma/seed.mongo.ts

cd ../..
دسته‌بندی‌های سید‌شده: اسکریپت سید چهار دسته اصلی می‌سازد: سینما ترس (cinema-horror)، بردگیم (board-game)، مافیا (mafia) و لیزرتگ (lasertag).

بررسی داده‌ها

docker compose exec mongo mongosh tiktakrun_db -u tiktakrun -p --authenticationDatabase admin \
  --eval "db.categories.find().pretty()"

۹. تنظیم Nginx + WebSocket

Nginx به‌عنوان Reverse Proxy جلوی سه اپ قرار می‌گیرد. توجه ویژه به تنظیمات WebSocket (Socket.io) برای API:

# /etc/nginx/sites-available/tiktakrun

# --- سایت کاربران ---
server {
    listen 80;
    server_name example.com www.example.com;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# --- داشبورد ادمین ---
server {
    listen 80;
    server_name admin.example.com;
    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# --- API + WebSocket ---
server {
    listen 80;
    server_name api.example.com;
    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;          # برای WebSocket
        proxy_set_header Connection "upgrade";           # برای WebSocket
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;                        # اتصال طولانی WS
    }
}
# فعال‌سازی
sudo ln -s /etc/nginx/sites-available/tiktakrun /etc/nginx/sites-enabled/
sudo nginx -t            # تست صحت کانفیگ
sudo systemctl reload nginx
کلیدی برای چت/نوتیفیکیشن زنده: بدون دو هدر Upgrade و Connection "upgrade"، اتصال Socket.io کار نمی‌کند و چت زنده و اعلان‌ها از کار می‌افتند.

۱۰. SSL با Certbot و Let's Encrypt

# گرفتن گواهی برای هر سه دامنه
sudo certbot --nginx -d example.com -d www.example.com -d admin.example.com -d api.example.com

# تست تمدید خودکار
sudo certbot renew --dry-run
Certbot به‌صورت خودکار بلاک‌های server در Nginx را برای HTTPS (پورت 443) بازنویسی می‌کند و ریدایرکت HTTP→HTTPS اضافه می‌کند. تمدید گواهی توسط systemd timer انجام می‌شود.

۱۱. اتصال داشبورد ادمین به بک‌اند

داشبورد ادمین (@tiktakrun/admin) از طریق یک axios client به API متصل می‌شود. نکات کلیدی اتصال:

موضوعمقدار / مسیر
آدرس پایه APINEXT_PUBLIC_API_URL → مثلاً https://api.example.com
توکن دسترسی (canonical)localStorage["ttr_admin_access_token"]
axios clientsrc/lib/api/client.ts (با getAccessToken())
ساختار پاسخ{ success: true, data: { data, total, page, limit } }
ResponseInterceptor: یک interceptor سراسری در API همه پاسخ‌ها را در { success, data } می‌پیچد. در صفحات ادمین یک تابع unwrap() داریم که در صورت وجود res.data.data آن را باز می‌کند.

۱۲. پشتیبان‌گیری خودکار (mongodump)

اسکریپت‌های آماده در infra/scripts/ قرار دارند. پشتیبان‌گیری با mongodump به‌صورت آرشیو فشرده انجام می‌شود:

# پشتیبان‌گیری دستی
bash infra/scripts/backup.sh
# داخل اسکریپت:
mongodump --archive=backup-$(date +%F).gz --gzip \
  --uri="mongodb://tiktakrun:PASS@localhost:27017/tiktakrun_db?authSource=admin&replicaSet=rs0"

# بازیابی
mongorestore --archive=backup-2026-06-18.gz --gzip --drop \
  --uri="mongodb://tiktakrun:PASS@localhost:27017/?authSource=admin&replicaSet=rs0"

زمان‌بندی خودکار با Cron

# ویرایش crontab
crontab -e

# هر روز ساعت ۳ بامداد پشتیبان بگیر
0 3 * * * /opt/tiktakrun/infra/scripts/backup.sh >> /var/log/ttr-backup.log 2>&1
